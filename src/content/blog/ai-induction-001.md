## 1. Java 集合框架主要由哪些部分组成？

【面试题】  
Java 集合框架主要由哪些部分组成？`Collection` 和 `Map` 有什么区别？

【核心答案】  
Java 集合框架主要分为两大体系：

- `Collection`：单列集合，用来存储一个一个的对象。
- `Map`：双列集合，用来存储键值对 `key-value`。

`Collection` 下面又分为三类：

- `List`：元素有序、可重复。
- `Set`：元素不可重复，通常无序。
- `Queue`：通常按照先进先出规则组织元素。

【原理解析】  
`Collection` 更关注“元素本身”的管理，而 `Map` 更关注“键和值之间映射关系”的管理。  
从使用方式看，`List` 适合下标访问和顺序遍历，`Set` 适合去重，`Queue` 适合缓冲和调度，`Map` 适合按键快速查找值。

【面试扩展】  
- 常见 `List` 实现类：`ArrayList`、`LinkedList`、`Vector`。
- 常见 `Set` 实现类：`HashSet`、`LinkedHashSet`、`TreeSet`。
- 常见 `Queue` 实现类：`PriorityQueue`、`ArrayDeque`。
- 常见 `Map` 实现类：`HashMap`、`LinkedHashMap`、`Hashtable`、`TreeMap`、`ConcurrentHashMap`。

【一句话总结】  
Java 集合框架本质上就是“单列结构用 `Collection`，键值结构用 `Map`”。

## 2. List、Set、Queue 分别有什么特点？

【面试题】  
`List`、`Set`、`Queue` 三者的核心区别是什么？

【核心答案】  
- `List`：有序、可重复、支持按索引访问。
- `Set`：不可重复，通常不保证插入顺序。
- `Queue`：一般遵循先进先出，适合排队和任务处理。

【原理解析】  
三者的差异主要来自设计目标不同：

- `List` 强调“顺序”和“位置”。
- `Set` 强调“唯一性”。
- `Queue` 强调“进入和移除的先后规则”。

所以在选型时，先看业务需要的是“顺序”“去重”还是“排队处理”。

【面试扩展】  
- `PriorityQueue` 是特殊队列，出队顺序由优先级决定，不一定是 FIFO。
- `LinkedHashSet` 在 `Set` 基础上保留插入顺序。
- `TreeSet` 可以对元素自动排序。

【一句话总结】  
`List` 管顺序，`Set` 管去重，`Queue` 管出入队规则。

## 3. ArrayList 和 Vector 有什么区别？

【面试题】  
`ArrayList` 和 `Vector` 都是基于数组实现，它们有什么区别？

【核心答案】  
两者主要区别有三个：

- `ArrayList` 线程不安全，`Vector` 线程安全。
- `ArrayList` 扩容通常为原容量的 `1.5` 倍，`Vector` 扩容通常为原容量的 `2` 倍。
- `ArrayList` 现在使用更普遍，`Vector` 属于较老的同步容器。

【原理解析】  
`Vector` 的很多方法都加了 `synchronized`，因此能保证多线程下的基本安全，但也带来了额外的同步开销。  
`ArrayList` 没有内置同步，单线程场景性能通常更好。  
扩容策略不同会影响内存利用率和扩容次数：`1.5` 倍更节省空间，`2` 倍扩容更激进。

【面试扩展】  
- 多线程下如果仍想用动态数组，通常更推荐 `Collections.synchronizedList(...)` 或并发容器方案，而不是直接使用 `Vector`。
- 面试中如果被问“为什么现在很少用 `Vector`”，核心就是“历史包袱大，线程安全方式粗粒度，性能和灵活性都不够理想”。

【一句话总结】  
`ArrayList` 是现代开发默认选择，`Vector` 主要是早期同步容器的历史产物。

## 4. ArrayList、Vector、LinkedList 的底层实现分别是什么？

【面试题】  
`ArrayList`、`Vector`、`LinkedList` 的底层数据结构是什么？

【核心答案】  
- `ArrayList`：动态数组。
- `Vector`：动态数组。
- `LinkedList`：双向链表。

【原理解析】  
数组结构支持按下标快速定位，因此查询效率高；但中间插入、删除时需要移动元素。  
链表结构节点分散存储，每个节点记录前驱和后继地址，因此插入、删除时不需要大规模搬移元素，但随机访问性能较差。

【面试扩展】  
- `LinkedList` 既可以当 `List` 用，也可以当 `Queue` 或 `Deque` 用。
- 现代 JVM 和 CPU 缓存机制下，`ArrayList` 在很多场景下实际表现往往优于 `LinkedList`，即使是部分插入删除场景也未必链表更快。

【一句话总结】  
`ArrayList/Vector` 背后是数组，`LinkedList` 背后是双向链表。

## 5. ArrayList 和 LinkedList 有什么区别？如何选型？

【面试题】  
`ArrayList` 和 `LinkedList` 有什么区别？实际开发中应该怎么选？

【核心答案】  
主要区别如下：

- 底层结构不同：`ArrayList` 是数组，`LinkedList` 是双向链表。
- 随机访问不同：`ArrayList` 支持快速按索引访问，`LinkedList` 较慢。
- 插入删除不同：`LinkedList` 在已知节点位置时更适合插入删除，`ArrayList` 可能需要移动元素。
- 内存占用不同：`LinkedList` 每个节点要额外保存前后指针，内存开销更大。

实际开发中，`ArrayList` 更常用，只有在频繁头尾插入删除、并且确实需要链表特性时才考虑 `LinkedList`。

【原理解析】  
`ArrayList` 的优势来自连续内存空间，读取快、缓存友好；  
`LinkedList` 的优势来自节点链接，结构修改灵活。  
但很多人只记住“链表增删快”，忽略了前提是“已经定位到节点”。如果先要遍历找到位置，总代价并不一定低。

【面试扩展】  
- `ArrayList` 默认构造时不会立刻分配大数组，首次添加元素时才真正分配容量。
- `LinkedList` 实现了 `Deque` 接口，适合双端队列场景。
- 如果面试官问“为什么业务代码里很少看到 `LinkedList`”，可以从缓存命中率、对象开销、随机访问差等角度回答。

【一句话总结】  
默认优先 `ArrayList`，`LinkedList` 只在明确需要链表特性时使用。

## 6. RandomAccess 接口有什么作用？

【面试题】  
`RandomAccess` 接口有什么用？为什么 `ArrayList` 实现了它而 `LinkedList` 没有？

【核心答案】  
`RandomAccess` 是一个标记接口，用来表明该集合支持快速随机访问。  
`ArrayList` 基于数组，按索引访问效率高，所以实现了它；`LinkedList` 基于链表，随机访问效率低，所以没有实现。

【原理解析】  
它本身没有定义方法，主要用于告诉框架和开发者：这个集合适合使用基于索引的遍历方式。  
例如某些通用算法会根据是否实现 `RandomAccess` 来决定采用普通 `for` 循环还是迭代器遍历。

【面试扩展】  
- 标记接口本质上是一种“能力声明”。
- 常见的相关追问是“`ArrayList` 遍历为什么常用 `for`，`LinkedList` 更适合迭代器”，这和底层结构及 `RandomAccess` 标记有关。

【一句话总结】  
`RandomAccess` 不提供能力，只负责声明“我适合随机访问”。

## 7. ArrayList 扩容机制是怎样的？

【面试题】  
`ArrayList` 的扩容机制是什么？

【核心答案】  
`ArrayList` 底层是数组，数组容量固定，空间不够时会触发扩容。  
默认情况下，新容量通常扩为旧容量的 `1.5` 倍，然后把旧数组数据复制到新数组中。

【原理解析】  
扩容本质是“申请更大的新数组 + 拷贝旧数据”，因此扩容本身是有成本的。  
这也是为什么如果能预估元素个数，提前指定初始容量会更高效，可以减少多次扩容和数组拷贝。

【面试扩展】  
- `ArrayList` 并不是每次 `add` 都扩容，只有容量不足时才扩容。
- 面试常问“为什么是 `1.5` 倍而不是 `2` 倍”，回答重点是空间和扩容次数之间的折中。
- 如果频繁扩容，可能导致性能抖动。

【一句话总结】  
`ArrayList` 通过“按需扩容 + 数组复制”实现动态增长。

## 8. HashSet、LinkedHashSet、TreeSet 有什么区别？

【面试题】  
`HashSet`、`LinkedHashSet`、`TreeSet` 的区别是什么？

【核心答案】  
- `HashSet`：基于哈希表实现，元素无序，不保证遍历顺序。
- `LinkedHashSet`：在 `HashSet` 基础上维护插入顺序。
- `TreeSet`：基于排序结构实现，元素默认按自然顺序或自定义比较器排序。

【原理解析】  
三者共同点是都属于 `Set`，都强调元素唯一。  
不同点在于“如何组织元素”：

- `HashSet` 关注查重和查找效率。
- `LinkedHashSet` 在哈希结构之外维护顺序信息。
- `TreeSet` 通过有序结构维护排序结果。

因此它们分别适合“只去重”“既去重又保序”“去重且排序”三类场景。

【面试扩展】  
- `TreeSet` 放入元素时要求元素具备可比较性，通常依赖 `Comparable` 或 `Comparator`。
- 如果面试官追问“为什么 `Set` 能去重”，通常会继续问到 `hashCode()` 和 `equals()` 的配合机制。

【一句话总结】  
三种 `Set` 的核心差异就是“无序、保序、排序”。

## 9. PriorityQueue 是什么？有什么特点？

【面试题】  
`PriorityQueue` 和普通队列有什么区别？

【核心答案】  
`PriorityQueue` 是优先级队列，出队顺序不一定遵循先进先出，而是按照元素优先级决定谁先出队。  
默认情况下，它会按元素的自然顺序构建小顶堆。

【原理解析】  
普通队列强调“谁先进谁先出”，而优先级队列强调“谁优先级高谁先出”。  
因此它更适合任务调度、定时处理、TopK、最小值/最大值维护等场景。

【面试扩展】  
- `PriorityQueue` 底层通常使用堆实现。
- 如果放入自定义对象，往往需要实现比较规则。
- 它不允许插入 `null` 元素。

【一句话总结】  
`PriorityQueue` 不是按入队顺序出队，而是按优先级出队。

## 10. 什么是无界队列和有界队列？

【面试题】  
无界队列和有界队列有什么区别？

【核心答案】  
- 无界队列：理论上容量无限，直到系统资源耗尽。
- 有界队列：容量固定，达到上限后再插入会失败、阻塞或触发特定策略。

【原理解析】  
两者的核心区别是是否对队列容量做明确限制。  
无界队列使用方便，但如果生产速度远高于消费速度，容易导致内存不断膨胀。  
有界队列可以起到限流和削峰作用，更利于系统稳定性控制。

【面试扩展】  
- 线程池参数设计时，工作队列是否有界是非常关键的系统设计点。
- 面试中可以补充“有界队列更适合生产环境的资源控制”。

【一句话总结】  
无界队列风险在内存失控，有界队列价值在容量治理。

## 11. 什么是 BlockingQueue？为什么它在线程池中很重要？

【面试题】  
什么是阻塞队列 `BlockingQueue`？它有什么作用？

【核心答案】  
`BlockingQueue` 是支持阻塞操作的队列，常用于多线程生产者-消费者模型。  
当队列为空时，获取元素的线程可以阻塞等待；当队列已满时，插入元素的线程也可以阻塞等待。

【原理解析】  
它在普通队列基础上加入了线程协作能力，能够自动协调“生产速度”和“消费速度”，避免开发者自己手写复杂的等待通知逻辑。  
因此它是并发编程中的基础组件，也是线程池工作队列的重要抽象。

【面试扩展】  
- 常见实现有 `ArrayBlockingQueue`、`LinkedBlockingQueue`、`PriorityBlockingQueue`。
- `BlockingQueue` 常见方法分为抛异常、返回特殊值、一直阻塞、超时等待四类。
- 在线程池中，任务提交后通常会先进入阻塞队列等待工作线程处理。

【一句话总结】  
`BlockingQueue` 的核心价值是让线程安全地“等得到、放得下”。

## 12. BlockingQueue 的实现类有哪些？各自适合什么场景？

【面试题】  
常见的 `BlockingQueue` 实现类有哪些？它们分别有什么特点？

【核心答案】  
常见实现类如下：

- `ArrayBlockingQueue`：基于数组实现的有界阻塞队列，容量固定。
- `LinkedBlockingQueue`：基于链表实现的阻塞队列，可选有界，也可近似看作无界。
- `PriorityBlockingQueue`：支持优先级排序的无界阻塞队列。
- `DelayQueue`：支持延迟获取元素的无界阻塞队列。

【原理解析】  
不同实现的本质差异来自底层结构和出队规则：

- 数组结构更紧凑，适合容量可控场景。
- 链表结构扩展性更强，适合吞吐量较大的场景。
- 优先级队列适合按权重调度。
- 延迟队列适合定时任务和过期处理。

【面试扩展】  
- 线程池默认常见的工作队列之一就是 `LinkedBlockingQueue`。
- `DelayQueue` 常被用于缓存过期、订单超时关闭、定时重试等场景。
- `PriorityBlockingQueue` 虽然会阻塞获取，但队列本身通常是无界的，因此仍需注意内存风险。

【一句话总结】  
选择哪种阻塞队列，本质上取决于“容量限制、顺序规则、业务调度需求”。

## 13. 为什么说集合选型比死记 API 更重要？

【面试题】  
Java 集合面试中，为什么经常强调“理解选型”而不是只背 API？

【核心答案】  
因为面试官真正想考察的是：你是否能根据业务特征，选择合适的数据结构。  
集合的本质不是 API 记忆，而是时间复杂度、空间开销、线程安全、顺序要求、唯一性要求等权衡。

【原理解析】  
例如：

- 需要随机访问，优先 `ArrayList`。
- 需要去重且不关心顺序，优先 `HashSet`。
- 需要保留插入顺序，考虑 `LinkedHashSet` 或 `LinkedHashMap`。
- 需要排序，考虑 `TreeSet`、`TreeMap` 或 `PriorityQueue`。
- 需要线程协作，考虑 `BlockingQueue`。

这类回答比单纯背诵类名更能体现工程理解。

【面试扩展】  
- 回答集合题时，最好始终围绕“底层结构 + 特性 + 场景 + 代价”四个角度展开。
- 如果能补充并发、内存、扩容、排序、去重机制，回答会更完整。

【一句话总结】  
集合题的本质不是背类名，而是做数据结构选型。
